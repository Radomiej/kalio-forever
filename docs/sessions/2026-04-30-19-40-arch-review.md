## Phase 5 — Prioritized Recommendations

### 🔴 Immediate (before next major feature)

- [ ] **Fix `memory.tools.spec.ts` syntax error** (line 113) — 1 test file broken, prevents full coverage run
- [ ] **Split `ChatInterface.tsx` (547 LOC)** — hardest limit violation; extract `useRaAppBridge`, `useChatSubscriptions`, `ChatInputArea` as co-located sub-files
- [ ] **Split `@kalio/types/src/index.ts`** — create domain barrel files (`types/src/chat.ts`, `types/src/raapp.ts`, etc.) re-exported from `index.ts`; currently at 464 LOC, will hit 600+ within 2 features
- [ ] **Fix `LLMPanel.tsx:147` silent catch** — replace `.catch(() => null)` with logged error or typed result
- [ ] **Fix `useContextUsage.ts:42,51` `any` casts** — replace with typed `AgentStore` selector
- [ ] **Add `requiresConfirmation: true` audit** — verify all `raapp` tools that trigger destructive native systems propagate confirmation correctly (currently `run_raapp` is `false`)

### 🟡 Short-term (next sprint)

- [ ] **Break `tool-registry.service.ts → subagent.tool.ts` circular dep** — extract shared interface
- [ ] **Add coverage for `modules/raapp`** — currently 23% — add unit tests for `EffectsProcessorService`, `RAAppHITLService`, `NativeSystemRegistry`
- [ ] **Add coverage for `modules/mcp`** — currently 0% — even smoke-level tests needed
- [ ] **Add VFS lifecycle policy** — TTL-based session file cleanup, or manual archive trigger
- [ ] **Make agentic iteration limit configurable per persona** — move magic `8` to `PersonaSessionConfig`
- [ ] **Add `/api/health` endpoint** — needed for deployment monitoring and `start-dev.ps1` readiness probe
- [ ] **Scope MCP tools per persona** — filter available MCP tools in `PersonaSessionConfig.availableSkills`
- [ ] **Delete 6 dead exports** (knip findings) — reduces bundle and confusion

### 🟢 Long-term (roadmap)

- [ ] **API key encryption** — implement `libsodium secretbox` as noted in schema comments
- [ ] **Socket.IO rate limiting** — add throttle guard on `chat:send` (e.g., max 3 concurrent per session)
- [ ] **Socket.IO Redis adapter** — needed before horizontal scaling
- [ ] **KalioSDK reconnect/resume** — handle mid-stream WebSocket drops with exponential backoff
- [ ] **Remote Turborepo cache** — speed up CI builds
