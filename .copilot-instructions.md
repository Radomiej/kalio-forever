# Copilot Instructions for Kalio Project

## Required Reading

**ALWAYS read `AGENTS.md` before starting any task.** This file contains critical project-specific rules, conventions, and constraints that you must follow.

## Project Overview

Kalio is a TypeScript monorepo with:
- **Backend**: NestJS 11 API (apps/kalio-api/src/)
- **Frontend**: React 19 web app (apps/kalio-web/src/)
- **Shared packages**: packages/@kalio/
- **Package manager**: pnpm 9 with Turborepo 2.4

## Key Commands

- Install: `pnpm install` (from root)
- Build: `pnpm turbo run build`
- Test: `pnpm turbo run test`
- Lint: `pnpm turbo run lint`
- Typecheck: `pnpm turbo run typecheck`
- Run locally: `.\start-dev.ps1`

## Critical Constraints

- **Cross-module imports**: Only import from `@kalio/types` across module boundaries
- **No `any` types**: Use `unknown` + narrowing or explicit types
- **No empty catch blocks**: Always log errors with context
- **LLM calls**: All LLM traffic goes through Socket.IO gateway (never from frontend)
- **File I/O**: All filesystem access through VFSService only
- **Types**: All shared types live in `@kalio/types/src/index.ts`

## Testing

- Unit/integration: Vitest
- E2E: Playwright (apps/e2e/tests/)
- Mock LLM with `MockLLMProvider` in tests

## Do Not Modify

- `packages/@kalio/types/**` (contract changes require PR review)
- `apps/kalio-api/src/main.ts` (bootstrap only)
- `turbo.json`, `pnpm-workspace.yaml`, `drizzle.config.ts`

## Before Making Changes

1. Read AGENTS.md completely
2. Read the files you will touch
3. Read files that call the files you will touch
4. Match existing patterns in the codebase
5. State your plan before editing

## Style Conventions

- PascalCase for classes
- camelCase for variables/methods
- kebab-case for files
- Match project's existing indentation, quotes, naming
